# Support

## Where to ask questions

- GitHub Issues: use issues for bugs, feature requests, and docs problems.

## What to include in a bug report

- Node.js version
- Operating system
- A minimal reproduction (small script or repository)
- What you expected vs what happened

## Security issues

If you believe you’ve found a security vulnerability, avoid posting it publicly
in an issue. Reach out to the maintainer directly (see the repository profile)
or open a private advisory if the repository supports it.
